Markdown là một ngôn ngữ đánh dấu nhẹ (lightweight markup language) được tạo ra bởi John Gruber vào năm 2004. Mục đích của Markdown là giúp viết văn bản có cấu trúc một cách dễ đọc và dễ viết, sau đó có thể chuyển đổi sang HTML hoặc các format khác.

💡 Tại sao nên học Markdown?
  • ✅ Đơn giản, dễ học (học trong 15 phút)
  • ✅ Được sử dụng rộng rãi (GitHub, Reddit, Discord, Notion...)
  • ✅ Tập trung vào nội dung, không phải định dạng
  • ✅ Có thể convert sang PDF, HTML, DOCX...
  • ✅ Version control friendly (dùng được với Git)

1️⃣ Cú Pháp Cơ Bản

Tiêu Đề (Headers)

Markdown hỗ trợ 6 cấp độ tiêu đề, từ H1 (lớn nhất) đến H6 (nhỏ nhất):

# H1 - Tiêu đề cấp 1 (Lớn nhất)
## H2 - Tiêu đề cấp 2
### H3 - Tiêu đề cấp 3
#### H4 - Tiêu đề cấp 4
##### H5 - Tiêu đề cấp 5
###### H6 - Tiêu đề cấp 6 (Nhỏ nhất)
💡 Best Practice:
  • Chỉ dùng 1 H1 duy nhất cho cả document (tiêu đề chính)
  • H2 cho các sections lớn
  • H3, H4 cho subsections
  • Luôn có khoảng trống sau dấu #

2️⃣ Định Dạng Text

Basic Formatting

**Bold text** hoặc __Bold text__
*Italic text* hoặc _Italic text_
***Bold và Italic*** hoặc ___Bold và Italic___
~~Strikethrough~~ (Gạch ngang)
`Inline code` (Code ngắn)

Kết quả:

  • Bold text
  • Italic text
  • Bold và Italic
  • Strikethrough
  • Inline code

Advanced Formatting

<u>Underline text</u> (Dùng HTML)
<mark>Highlighted text</mark>
H<sub>2</sub>O (Subscript)
X<sup>2</sup> (Superscript)
💡 Pro Tips:
  • Dùng **bold** cho từ khóa quan trọng
  • Dùng *italic* cho nhấn mạnh nhẹ hoặc thuật ngữ nước ngoài
  • Dùng ~~strikethrough~~ cho nội dung đã cũ/deprecated

3️⃣ Danh Sách

Unordered List (Danh sách không thứ tự)

- Item 1
- Item 2
  - Sub-item 2.1
  - Sub-item 2.2
    - Sub-sub-item 2.2.1
- Item 3

Kết quả:

  • Item 1
  • Item 2
    • Sub-item 2.1
    • Sub-item 2.2
  • Item 3

Ordered List (Danh sách có thứ tự)

1. Bước 1
2. Bước 2
   1. Bước 2.1
   2. Bước 2.2
3. Bước 3

Kết quả:

  1. Bước 1
  2. Bước 2
    1. Bước 2.1
    2. Bước 2.2
  3. Bước 3
💡 Best Practice:
  • Indent sub-items bằng 2 hoặc 4 spaces
  • Dùng ordered list cho các bước tuần tự
  • Dùng unordered list cho các items không theo thứ tự

4️⃣ Links & Images

Links

[Text hiển thị](https://example.com)
[Link với title](https://example.com "Title khi hover")
[Reference link][1]

[1]: https://example.com "Định nghĩa reference ở cuối"

<https://auto-link.com> (Tự động tạo link)

Images

![Alt text](https://via.placeholder.com/300)
![Image với title](https://via.placeholder.com/300 "Title")

Hoặc dùng reference:
![Alt text][logo]

[logo]: https://via.placeholder.com/150 "Logo"

Images với Link

[![Alt text](image-url)](link-url)
💡 Pro Tips:
  • Luôn có alt text cho accessibility
  • Dùng reference links cho links lặp lại nhiều lần
  • Dùng relative paths cho images trong cùng repository

5️⃣ Code Blocks

Inline Code

Dùng `console.log()` để debug code.

Kết quả: Dùng console.log() để debug code.

Code Block với Syntax Highlighting

```javascript
function greet(name) {
  console.log(`Hello, ${name}!`);
}
greet("World");
```

```python
def greet(name):
    print(f"Hello, {name}!")
    
greet("World")
```

```css
.container {
  max-width: 1200px;
  margin: 0 auto;
}
```
💡 Pro Tips:
  • Luôn chỉ rõ ngôn ngữ để có syntax highlighting
  • Supported languages: javascript, python, java, css, html, bash, sql, json, yaml, markdown...
  • Dùng 3 backticks (```) không 4 để tránh lỗi

6️⃣ Bảng

Basic Table

| Header 1 | Header 2 | Header 3 |
|----------|----------|----------|
| Row 1    | Data     | Data     |
| Row 2    | Data     | Data     |

Kết quả:

Header 1 Header 2 Header 3
Row 1 Data Data
Row 2 Data Data

Table với Alignment

| Left Align | Center Align | Right Align |
|:-----------|:------------:|------------:|
| Left       | Center       | Right       |
| Text       | Text         | 1000        |

Kết quả:

Left Align Center Align Right Align
Left Center Right
Text Text 1000
💡 Best Practice:
  • :--- = Left align (default)
  • :---: = Center align
  • ---: = Right align
  • Dùng emojis để table dễ đọc hơn
  • Align numbers về bên phải

1️⃣3️⃣ Best Practices

✅ Structure

  1. Một H1 duy nhất (tiêu đề chính)
  2. Hierarchy rõ ràng: H1 → H2 → H3 (không skip levels)
  3. Dòng trống: Thêm dòng trống giữa các sections
  4. 80-100 characters/line cho readability

✅ Formatting

✅ GOOD:
  • Clear spacing
  • Consistent formatting
  • Proper hierarchy
❌ BAD:
  • No spacing
  • Inconsistent style
  • Poor hierarchy

✅ Links

✅ GOOD:
[Descriptive Link Text](url)

❌ BAD:
[Click here](url)
[Link](url)

✅ Tables

✅ GOOD:
| Name | Age | City |
|------|----:|------|
| John |  25 | NYC  |

❌ BAD:
|Name|Age|City|
|---|---|---|
|John|25|NYC|
(Khó đọc, không alignment)

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| page | integer | No | Page number |
| limit | integer | No | Items per page |

```json

  "users": [

```
